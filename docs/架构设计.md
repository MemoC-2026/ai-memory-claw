## 4. 关键算法设计

### 4.1 相似度搜索算法

```typescript
// 搜索流程
async function searchSimilar(query: string, limit: number, threshold: number) {
  // 1. 生成查询向量
  const queryVector = await embeddingModel.embed(query);
  
  // 2. ChromaDB 向量搜索
  const results = await chromaDB.query(
    queryVector, 
    limit * 3  // 多取一些用于过滤
  );
  
  // 3. 过滤低相关度
  const filtered = results.filter(r => r.score >= threshold);
  
  // 4. 排序返回
  return filtered
    .sort((a, b) => b.score - a.score)
    .slice(0, limit);
}
```

### 4.2 遗忘算法

```typescript
// 遗忘检查流程
function checkForgotten() {
  const candidates = [];
  
  for (const memory of memories) {
    const daysSinceAccess = getDaysSince(memory.lastAccessedAt);
    const importanceScore = getImportanceScore(memory.importance);
    
    // 遗忘条件
    if (daysSinceAccess > 30 && importanceScore < 0.3) {
      candidates.push(memory);
    } else if (daysSinceAccess > 60 && importanceScore < 0.5) {
      candidates.push(memory);
    } else if (daysSinceAccess > 90) {
      candidates.push(memory);
    }
  }
  
  return candidates;
}
```

// 记忆整合流程
async function integrate() {
  const clusters = discoverClusters();

### 5.2 钩子处理

```typescript
// before_agent_start 钩子
async function handleBeforeAgentStart(event: AgentStartEvent) {
  const config = getConfig();
  if (!config.autoRecall) return;
  
  // 1. 获取用户输入
  const prompt = event.prompt;
  
  // 2. 搜索相关记忆
  const memories = await memorySystem.search(prompt, {
    limit: config.recallLimit,
    threshold: config.recallThreshold
  });
  
  if (memories.length === 0) return;
  
  // 3. 格式化注入
  const context = formatMemoriesContext(memories);
  
  return {
    prependContext: context
  };
}

// agent_end 钩子
async function handleAgentEnd(event: AgentEndEvent) {
  const config = getConfig();
  if (!config.autoCapture) return;
  
  // 1. 生成对话摘要
  const summary = generateConversationSummary(event.messages);
  
  // 2. 分析话题分类
  const { category, subCategory } = analyzeCategory(summary);
  
  // 3. 判断重要性
  const importance = triggerAnalyzer.analyze(summary);
  
  // 4. 强制存储 (100%)
  await memorySystem.store({
    content: summary,
    category,
    subCategory,
    importance
  });
}
```

## 6. 错误处理设计

### 6.1 容错机制

| 场景 | 处理策略 |
|------|----------|
| ChromaDB 未启动 | 使用本地向量搜索备选 |
| 存储失败 | 重试3次，失败则记录日志 |
| 向量生成失败 | 使用关键词匹配备选 |
| 配置读取失败 | 使用默认值 |

### 6.2 日志分级

| 级别 | 使用场景 |
|------|----------|
| DEBUG | 详细调试信息 |
| INFO | 正常运行信息 |
| WARN | 可恢复的异常 |
| ERROR | 需要关注的错误 |

## 9. 性能优化

### 9.1 缓存策略

```
记忆访问缓存:
- LRU 缓存热点记忆
- 缓存大小: 100 条
- TTL: 1 小时

向量缓存:
- 缓存已编码的文本
- 缓存大小: 500 条
- TTL: 24 小时
```

### 9.2 异步处理

```
非阻塞操作:
- 摘要生成 → 异步
- 遗忘检查 → 定时任务
- 记忆整合 → 后台任务

阻塞操作:
- 记忆存储 (必须同步)
- 记忆搜索 (需要同步返回)
```
