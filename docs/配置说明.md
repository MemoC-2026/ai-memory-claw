# AI Memory Claw - 配置说明

> 文档版本：1.0  
> 创建日期：2026-03-28  
> 项目：AI 渐进式记忆系统 OpenClaw 插件

---

## 1. 配置概述

AI Memory Claw 通过 `openclaw.plugin.json` 文件进行配置。

---

## 2. 配置项详解

### 2.1 基础配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `dataDir` | string | `~/.ai-memory-claw/data` | 记忆数据存储目录 |

**说明**：
- 可以是绝对路径或相对路径
- 建议使用绝对路径确保一致性
- 目录会在首次运行时自动创建

**示例**：
```json
{
  "dataDir": "D:/ai-memory-claw/data"
}
```

---

### 2.2 自动运行配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `autoRecall` | boolean | `true` | 对话前自动召回相关记忆 |
| `autoCapture` | boolean | `true` | 对话后自动生成记忆 |
| `captureStrategy` | string | `"always"` | 存储策略: `always`(每次) 或 `selective`(关键词) |

**说明**：
- `autoRecall`: 开启后，每次用户发送消息，AI 回答前会自动搜索相关记忆并注入上下文
- `autoCapture`: 开启后，每次对话结束会自动生成记忆存储
- `captureStrategy`:
  - `always`: 每次对话都生成记忆（推荐，实现真正的无感运行）
  - `selective`: 只有关键词匹配时才生成记忆

---

### 2.3 召回配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `recallThreshold` | number | `0.3` | 召回相似度阈值 (0-1) |
| `recallLimit` | number | `3` | 单次召回记忆数量上限 |

**说明**：
- `recallThreshold`: 低于此相似度的记忆不会被召回
  - 0.0-0.3: 宽松匹配，召回更多
  - 0.3-0.5: 适中（推荐）
  - 0.5-1.0: 严格匹配，召回更少
- `recallLimit`: 每次召回的记忆数量上限，影响上下文长度

**示例**：
```json
{
  "recallThreshold": 0.35,
  "recallLimit": 5
}
```

---

### 2.4 捕获配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `captureMaxChars` | number | `500` | 单条记忆最大字符数 |

**说明**：
- 超过此长度的内容会被截断
- 过长的记忆会影响搜索性能

---

### 2.5 后处理配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enableSummary` | boolean | `true` | 长记忆自动生成摘要 |
| `enableForget` | boolean | `true` | 启用遗忘机制 |
| `enableIntegration` | boolean | `true` | 启用记忆整合机制 |
| `forgetIntervalDays` | number | `7` | 遗忘检查间隔(天) |
| `integrationThreshold` | number | `0.8` | 相似度阈值，超过则整合 |

**说明**：
- `enableSummary`: 自动将长记忆压缩为摘要，节省存储空间
- `enableForget`: 自动删除低价值记忆，保持存储效率
- `enableIntegration`: 自动将相似记忆整合为记忆簇
- `forgetIntervalDays`: 每隔多少天运行一次遗忘检查
- `integrationThreshold`: 相似度超过此值则自动整合

## 3. 完整配置示例

### 3.1 开发测试配置

```json
{
  "dataDir": "./data",
  "autoRecall": true,
  "autoCapture": true,
  "captureStrategy": "always",
  "recallThreshold": 0.3,
  "recallLimit": 3,
  "captureMaxChars": 500,
  "enableSummary": true,
  "enableForget": false,
  "enableIntegration": false,
  "forgetIntervalDays": 7,
  "integrationThreshold": 0.8
}
```

**特点**：
- 开发环境禁用遗忘和整合，方便调试
- 较低的阈值，测试更全面

---

### 3.2 生产环境配置

```json
{
  "dataDir": "~/.ai-memory-claw/data",
  "autoRecall": true,
  "autoCapture": true,
  "captureStrategy": "always",
  "recallThreshold": 0.35,
  "recallLimit": 5,
  "captureMaxChars": 800,
  "enableSummary": true,
  "enableForget": true,
  "enableIntegration": true,
  "forgetIntervalDays": 3,
  "integrationThreshold": 0.85
}
```

**特点**：
- 生产环境启用所有优化
- 更高的阈值减少噪音
- 启用遗忘和整合保持效率

---

### 3.3 轻量配置（低资源）

```json
{
  "dataDir": "~/.ai-memory-claw/data",
  "autoRecall": true,
  "autoCapture": true,
  "captureStrategy": "always",
  "recallThreshold": 0.4,
  "recallLimit": 3,
  "captureMaxChars": 300,
  "enableSummary": true,
  "enableForget": true,
  "enableIntegration": false,
  "forgetIntervalDays": 1,
  "integrationThreshold": 0.8
}
```

**特点**：
- 禁用记忆整合节省资源
- 更频繁的遗忘检查
- 更严格的捕获长度

## 5. 关键词规则说明

### 5.1 重要性等级

关键词**不决定是否存储**，只决定 **importance 等级**：

| 等级 | 关键词示例 | 影响 |
|------|------------|------|
| **critical** | 记住、别忘了、密码、账号、银行卡 | 最高优先级，永不遗忘 |
| **high** | 决定、选择、采用、配置、安装、部署 | 高优先级，优先保留 |
| **medium** | 实现、修复、创建、完成 | 默认优先级 |
| **low** | 谢谢、你好、再见 | 低优先级，最先被遗忘 |

### 5.2 中文触发器

```
critical: 记住|别忘了|提醒我|密码|账号|身份|银行卡
high: 决定|选择|采用|配置|安装|部署
medium: 实现|修复|创建|帮助|完成|写|做
low: 你好|谢谢|再见|好的
```

### 5.3 英文触发器

```
critical: remember|don't forget|remind me|password|account|identity
high: decide|choose|use|adopt|config|install|deploy|setup
medium: implement|fix|create|help|complete|write|make|build
low: hello|thanks|bye|ok
```

## 6. 故障排除

### 6.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 记忆不召回 | `autoRecall: false` | 设为 `true` |
| 记忆不存储 | `autoCapture: false` | 设为 `true` |
| 召回太多无关 | `recallThreshold` 太低 | 调高到 0.4+ |
| 召回太少 | `recallThreshold` 太高 | 调低到 0.3 |
| 存储空间不足 | 遗忘未启用 | 启用 `enableForget` |

### 6.2 日志调试

开启详细日志：

```typescript
// 在插件代码中
api.logger.level = 'debug';
```

查看日志：

```bash
# OpenClaw 日志
openclaw logs
```

## 7. 配置迁移

### 7.1 备份配置

配置文件位于：
- Windows: `~/.ai-memory-claw/config.json`
- Linux/Mac: `~/.ai-memory-claw/config.json`

### 7.2 迁移数据

```bash
# 复制数据目录
cp -r ~/.ai-memory-claw/data /新路径/
```

然后更新 `dataDir` 配置。
