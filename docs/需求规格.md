# AI Memory Claw - 需求规格

> 文档版本：1.0  
> 创建日期：2026-03-28  
> 项目：AI 渐进式记忆系统 OpenClaw 插件

---

## 1. 项目概述

### 1.1 目标

构建一个真正无感运行的 AI 记忆系统，让 AI 拥有类似人类的记忆机制：
- **无感运行**：无需提示词，每次对话自动执行
- **强制记忆**：每次对话必然生成记忆，自然筛选
- **持久存储**：记忆存储在本地，不丢失
- **渐进强化**：常用记忆被强化，不常用被遗忘

### 1.2 核心理念

**量变→质变**：
- 不是"达到条件才存储"
- 而是"每次都存储，自然筛选"
- 记忆累积到一定程度，AI 自然形成"理解"和"灵魂"

## 2. 核心概念

### 2.1 记忆结构

```typescript
interface Memory {
  id: string;                    // 唯一标识，格式: CA-SB-YYYYMMDD-HHMM-XXXXXX
  taskDescription: string;        // 任务/对话描述
  
  // 分类
  category: string;              // 主分类
  subCategory: string;           // 子分类
  cluster: string;               // 记忆簇
  
  // 内容
  content: {
    task: string;               // 用户问题/任务
    process: string;            // AI 执行过程
    result: string;             // 执行结果
    insights: string[];          // 洞察/总结
  };
  
  // 向量嵌入 (用于相似度搜索)
  embedding: number[];
  
  // 时间戳
  createdAt: Date;
  updatedAt: Date;
  lastAccessedAt: Date;
  
  // 版本控制
  version: number;
  diffs: MemoryDiff[];
  mergedFrom: string[];
  
  // 评估指标
  confidence: number;            // 置信度 0-1
  usageCount: number;           // 访问次数
  importance: 'critical' | 'high' | 'medium' | 'low';
  
  // 标签
  tags: string[];
  
  // 加密 (可选)
  encrypted: boolean;
  encryptedContent?: string;
  
  // 自动清理
  autoCleanup?: boolean;
  cleanupAt?: Date;
  
  // 摘要
  summary?: string;
  originalSize?: number;
  compressedSize?: number;
}
```

### 2.2 记忆分类

| 分类 | 说明 | 示例 |
|------|------|------|
| 代码开发 | 编程、脚本、调试相关 | 帮用户写了 Python 脚本 |
| 系统运维 | 配置、部署、运维相关 | 帮用户配置了 Docker |
| 文档撰写 | 报告、文章、总结 | 帮用户写了需求文档 |
| 数据分析 | 数据处理、统计、可视化 | 帮用户分析了销售数据 |
| 创作设定 | 角色、故事、世界观 | 创建了哪吒角色设定 |
| 用户偏好 | 口味、习惯、偏好 | 用户喜欢川菜 |
| 日常事务 | 日程、提醒、杂事 | 用户问天气 |
| 其他 | 不属于上述类别 | 闲聊 |

### 2.3 记忆簇 (Memory Cluster)

相似记忆自然聚合成簇，形成更高层次的"理解"：

```typescript
interface MemoryCluster {
  id: string;
  name: string;

  subCategory: string;
  memories: string[];            // 记忆 ID 列表
  summary: string;               // 簇摘要
  createdAt: Date;
  updatedAt: Date;
}
```

## 3. 生命周期

```
新记忆 → 被访问(强化) → 长期不用(衰减) → 被遗忘/整合
```

### 3.1 访问强化

| 触发方式 | 强化效果 |
|----------|----------|
| 对话召回 | usageCount +1 |
| 手动查询 | usageCount +1 |
| 相似记忆整合 | importance 提升 |

### 3.2 遗忘阶段

| 阶段 | 条件 | 行为 |
|------|------|------|
| 活跃 | 创建 < 7 天 或 最近访问 | 正常响应 |
| 休眠 | 7-30 天未访问 | 降级到 L2/L3 |
| 归档 | 30+ 天未访问 | 降级到 L3 |
| 遗忘 | importance=low + 30天未访问 | 压缩或删除 |

## 4. 核心功能

### 4.1 自动召回 (Auto-Recall)

| 功能 | 说明 |
|------|------|
| 触发时机 | 每次用户发送消息，AI 回复前 |
| 触发钩子 | `before_agent_start` |
| 执行内容 | 搜索相关记忆，注入到 AI 上下文 |
| 相似度阈值 | 可配置 (默认 0.3) |
| 返回数量 | 可配置 (默认 3 条) |

### 4.2 自动捕获 (Auto-Capture)

| 功能 | 说明 |
|------|------|
| 触发时机 | 每次对话结束 |
| 触发钩子 | `agent_end` |
| 执行内容 | 强制生成记忆，存储到本地 |
| 存储策略 | 100% 存储，不依赖关键词 |
| 关键词 | 仅用于判断 importance 等级 |

### 4.3 强制记忆机制

```
每次 agent_end 必然执行:
1. 生成对话摘要
2. 分析话题分类
3. 提取关键实体 (如果有)
4. 判断用户反馈类型
5. 存储记忆 (100%)
```

### 4.4 自然筛选机制

| 机制 | 说明 | 触发时机 |
|------|------|----------|
| 遗忘 | 低优先级 + 长时间不用 → 遗忘 | 定时任务 |
| 整合 | 相似记忆 → 合并为簇 | 定时任务 / 存储时 |
| 强化 | 记忆被访问 → importance 提升 | 召回时 |
| 摘要 | 长记忆 → 自动生成摘要 | 存储时 |

### 4.5 启动欢迎 (方案A: 日志状态显示)

插件启动时，在日志中显示当前状态：

```
[AI Memory Claw] ✅ 插件已加载
[AI Memory Claw] 📚 当前记忆: 12 条
[AI Memory Claw] 🎯 自动召回: 已开启
[AI Memory Claw] 📝 自动捕获: 已开启
[AI Memory Claw] 🗂️  数据目录: ~/.ai-memory-claw/data
```

**说明**：
- 每次 OpenClaw 启动时自动显示
- 显示当前记忆数量
- 显示功能开关状态
- 显示数据目录路径

## 5. 关键词规则 (辅助功能)

关键词**不决定是否存储**，只决定 **importance 等级**：

| importance | 关键词 | 示例 |
|------------|--------|------|
| **critical** | 记住、别忘了、密码、账号、身份 | "记住我的密码是 xxx" |
| **high** | 决定、选择、采用、配置、安装、部署 | "决定使用 Docker" |
| **medium** | 实现、修复、创建、帮助、完成 | "帮我写了个脚本" |
| **low** | 闲聊、问候、感谢 | "谢谢"、"你好" |

## 6. OpenClaw 集成

### 6.1 生命周期钩子

| 钩子 | 用途 | 实现 |
|------|------|------|
| `before_agent_start` | 自动召回 | 搜索记忆，注入上下文 |
| `agent_end` | 自动捕获 | 强制生成记忆，存储 |

```json
  "name": "AI Memory Claw",

```

## 9. 验收标准

### 9.1 功能验收

| 编号 | 标准 | 验证方法 |
|------|------|----------|
| F1 | OpenClaw 启动时自动加载插件 | 检查日志 |
| F2 | 每次对话前自动搜索记忆 | 检查日志，查看注入内容 |
| F3 | 每次对话后自动生成记忆 | 查看数据目录 |
| F4 | 记忆可以在会话间持久保存 | 重启后查询 |
| F5 | 不重要的记忆会被遗忘 | 等待30天或触发遗忘 |
| F6 | 重要的记忆会被强化 | 多次召回同一记忆 |
| F7 | 相似记忆会被整合 | 查看记忆簇 |

### 9.2 性能验收

| 编号 | 标准 | 目标 |
|------|------|------|
| P1 | 记忆存储延迟 | < 100ms |
| P2 | 记忆检索延迟 | < 50ms |
| P3 | 启动时间 | < 3s |

### 9.3 体验验收

| 编号 | 标准 |
|------|------|
| E1 | 无需任何提示词，记忆系统自动运行 |
| E2 | 不需要告诉 AI "记住这个" |
| E3 | 对话感觉像有"记忆"的人 |
